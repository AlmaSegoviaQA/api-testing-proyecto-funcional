# API TESTING – Proyecto de Pruebas Funcionales
Proyecto desarrollado como parte del Bootcamp **QA Engineer – TripleTen**, enfocado en el análisis, diseño y ejecución de pruebas funcionales sobre endpoints REST.  
El objetivo principal fue validar respuestas del servidor, reglas de negocio, manejo de errores y consistencia de datos utilizando **Postman**.


## Objetivos del proyecto
- Validar el comportamiento de endpoints REST mediante solicitudes **POST, GET, PUT y DELETE**.  
- Diseñar y ejecutar **casos de prueba funcionales**, incluyendo pruebas positivas, negativas y de límites (boundary values).  
- Analizar códigos de estado HTTP y estructuras JSON.  
- Detectar, documentar y reportar defectos con pasos reproducibles y evidencias claras.  
- Garantizar consistencia entre los requisitos del sistema y los resultados obtenidos.


## Tecnologías y herramientas utilizadas
- **Postman** – Creación y ejecución de pruebas de API  
- **JSON** – Validación de estructuras y datos  
- **HTTP Methods & Status Codes**  
- **Jira** – Documentación y reporte de bugs  
- **Metodologías ágiles (Scrum)**  
- **Google Sheets** – Reporte estructurado del proyecto
